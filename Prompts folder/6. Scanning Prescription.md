### Scanning Prescription and Confirming Medicines After Scan  - Flow Structure

A camera-based scan flow that extracts medicine data from a prescription image, presents extracted results for user review and confirmation, and adds confirmed medicines to the list in bulk.

---

### Entry Point

**Home Screen** → **"+" Add button** (bottom right) → Select **"Scan Prescription"**

---

### Step 1 — Camera Screen

- Full screen camera view with a prescription frame/boundary overlay
- Icon: Flashlight toggle (top right), Image gallery picker (bottom left), Capture button (bottom center)
- Interaction: User aligns prescription within the frame and taps capture, or picks an existing photo from gallery
- After capture, a brief scanning animation plays with text "Reading your prescription…"

**Reasoning:** Full screen camera reduces distraction and keeps the user focused on aligning the prescription correctly. The boundary overlay guides non-tech-savvy users without any instruction text.

---

### Step 2 — Scan Result Review Screen

- Extracted medicines listed as individual cards (Medicine Name, Strength, Unit, Frequency if detected)
- Each card has an Edit icon (pencil) and a Remove icon (trash)
- Unrecognized or low-confidence fields are highlighted in amber with a warning icon
- User can tap any card to edit extracted details before confirming
- Option at bottom: **"+ Add medicine manually"** in case something was missed

**Reasoning:** Showing extracted results before confirming builds user trust. Users must verify AI output rather than blindly accepting it, which is critical in a health context where errors can be harmful.

---

### Step 3 — Edit Extracted Medicine (Conditional)

- Triggered only if user taps edit on a card
- Inline form opens: Medicine Name, Strength, Unit
- Confirm edit → returns to Step 2 review screen

**Reasoning:** Keeping edit inline rather than navigating to a new screen reduces friction and keeps the user in context of the full list while correcting a single item.

---

### Step 4 — Confirmation Screen

- Summary list of all medicines about to be added
- Total count shown at top: "X medicines will be added"
- Final CTA: **Confirm & Add All**
- Secondary action: **Go Back** to review screen

**Reasoning:** A dedicated confirmation screen before bulk adding prevents accidental additions. Since multiple medicines are added at once, a final checkpoint is more critical here than in manual single-medicine entry.

---

### Error States

- **Camera permission denied** — Full screen error illustration with message "Camera access is needed to scan prescriptions" and a CTA "Open Settings"
- **Scan failed — unreadable image** — Error message "We couldn't read this prescription. Please try again in better lighting or use a clearer image" with options "Retry Scan" and "Add Manually"
- **Scan failed — no medicine detected** — Message "No medicines found in this prescription" with options "Retry Scan" and "Add Manually"
- **Low confidence fields unresolved** — Inline amber warning on unresolved cards, confirmation CTA stays disabled with tooltip "Please review highlighted fields before confirming"
- **Network error during scan processing** — Message "Something went wrong. Check your connection and try again" with a "Retry" button
- **All medicines removed from list** — Confirm CTA disabled with message "No medicines to add. Add at least one or go back."

---

### Empty States

- **Gallery is empty** — If user opens gallery picker and has no photos, show message "No photos found in your gallery"
- **Scan result list becomes empty after all removed** — Illustration with message "You removed all medicines. Add at least one to continue" with CTA "Add Manually"

---

### Interaction Rules

- Capture button disabled until prescription frame is reasonably filled
- Low confidence fields must be resolved before proceeding to confirmation
- User cannot confirm if all medicines have been removed from the list
- Edited fields are marked with a small "Edited" tag for clarity
- If scan fails completely, app shows an error state with option to retry or switch to manual entry
- Gallery picker filters to image files only, no documents or videos

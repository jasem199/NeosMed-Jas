### Adding Medicine Manually - Flow Structure

A 3 step card flow followed by a final review screen. Each step groups logically related fields together. Cards are swipeable with auto-advance on completion. Ends with a "Looks good?" review screen where all details are visible and editable before final confirmation.

---

### Entry Point

**Home Screen** → Floating **"+" Add button** (bottom right) → Select **"Add Manually"**

---

### Step 1 — Medicine Details

- **Medicine Name** — Text input with auto-suggest dropdown from drug database as user types
- **Strength** — Numeric input (e.g. 500)
- **Unit** — Tap to select chip group (Pills / Syrup / Drop / Puff / Injection / Other)
- **Intake Advice** — Tap to select chip group (Before meal / After meal / With meal / No preference)

**Reasoning:** These four fields all describe what the medicine is and how it should be taken physically. Grouping them together feels natural — it mirrors how a user would read a medicine label from top to bottom.

---

### Step 2 — Schedule details

- **How many times per day?** — Tap to select chip group (1 / 2 / 3 / 4 / More)
- **Time pickers** — Generated dynamically based on frequency selected. Each intake gets a labeled drum-style time picker (First intake, Second intake…)
- **Duration** — Tap to select chip group (Ongoing / Specific days / Until further notice)
- If "Specific days" selected, a numeric day count input appears inline below

**Reasoning:** Frequency, time and duration are all schedule decisions. A user thinks about them together — "I take this twice a day, morning and night, for 7 days." Keeping them in one step mirrors that natural thought process.

---

### Step 3 — Stock & Notes

- **Current stock quantity** — Numeric input with unit label auto-filled from Step 1 (e.g. "pills")
- **Remind me when stock reaches** — Numeric input with helper text "We'll notify you to restock in time"
- **Reason / Note** — Free text input, 100 character limit, labeled as **(Optional)**

**Reasoning:** Stock and notes are supporting information, not core to the medicine schedule. Separating them into Step 3 prevents the earlier steps from feeling overwhelming. Making reason optional but visible here encourages users to add context without making it feel mandatory.

---

### Final Screen — Looks Good?

- Clean summary card showing all entered details grouped in the same 3 section structure
- Each field has a small **Edit** icon (pencil) next to it — tapping opens an inline edit mode for that field only
- Sections:
    - **Medicine** — Name, Strength, Unit, Intake Advice
    - **Schedule** — Frequency, Intake times, Duration
    - **Stock & Notes** — Stock quantity, Low stock threshold, Reason
- **Primary CTA: Confirm & Add Medicine**
- **Secondary action: Go Back**

**Reasoning:** A dedicated review screen before confirming builds user confidence especially for aged users who may have made an error mid-flow. Inline editing per field avoids sending the user all the way back to a previous step for a small correction.

---

### Error States

- **Medicine name empty** — Inline error below field "Medicine name is required" on attempting to advance
- **Invalid strength value** — Inline error "Please enter a valid numeric strength"
- **No unit selected** — Chip group highlighted with a shake, message "Please select a unit"
- **No frequency selected** — Step 2 advance blocked, chip group highlighted with message "Please select how many times per day"
- **Time not set for an intake** — Inline error below empty time picker "Please set a time for intake [number]"
- **Duplicate medicine detected** — Warning banner after name entry "This medicine already exists. Add again?" with options "Add Anyway" and "View Existing"
- **Invalid stock quantity** — Inline error "Please enter a valid number"
- **Low stock threshold higher than current stock** — Inline error "Reminder threshold cannot be higher than current stock"
- **Drug database unavailable** — Auto-suggest silently disabled, field works as plain text with no error shown

**Reasoning:** Errors appear only on attempt to advance, never while typing. This avoids making the user feel interrupted. Inline errors point directly to the problem field rather than a generic bottom toast.

---

### Empty States

- **Auto-suggest returns no results** — Dropdown shows "No medicine found. You can still type the name manually"
- **No unit selected on review screen** — Field shows a subtle "Not selected" placeholder in muted color

---

### Interaction Rules

- Progress indicator at top showing current step (1 of 3 + Review)
- Auto-advance to next step on completing all required fields in current step
- Left swipe and top left arrow icon button always allowed to go back to previous step
- Right swipe blocked until current step is fully complete
- Time pickers generated dynamically based on frequency — removed automatically if frequency is changed
- All fields on review screen are editable via inline pencil icon without navigating back
- Required fields marked with a subtle asterisk
- Optional fields labeled explicitly as "(Optional)"
- Keyboard dismisses on tapping outside input field
- On successful confirmation, user returns to home screen with a success toast "Medicine added successfully"
